Week 1
今天主要完成了 FastAPI 后端服务的基础搭建与接口测试。
1、完成 FastAPI 项目基础运行  
通过 uvicorn 启动 FastAPI 服务：
uvicorn app.main:app --reload
服务成功运行在：
http://127.0.0.1:8000
说明项目基本框架运行正常。

2、学习并使用 FastAPI 自动文档  
通过访问：http://127.0.0.1:8000/docs
可以进入 Swagger UI 页面，在页面中可以直接测试接口请求。之前对 /docs 的作用不太清楚，后来了解到这是 FastAPI 自动生成的接口文档，可以直接在网页中测试 API。

3、测试文件上传接口  
使用 /upload 接口上传文件。
上传成功后接口返回：
{
  "task_id": 1
}
说明文件上传后系统会生成一个任务记录，并返回 task_id。

4、新增任务查询接口  
为了查看任务处理情况，新增接口：
/tasks/{task_id}

用于根据 task_id 查询任务信息。
测试方式：
http://127.0.0.1:8000/tasks/1
成功返回任务记录，说明接口逻辑正常。

5、整体测试流程  
完整流程如下：
文件上传 → 返回 task_id → 使用 task_id 查询任务
该流程测试成功。
6、目前理解  
当前后端系统的基本结构为：
文件上传接口 → 创建任务记录 → 返回 taskID  
任务查询接口 → 根据ID查询任务状态
这一部分主要是为后续的文档解析模块做准备。

当前已完成文件上传与任务查询的基础流程搭建，为后续项目整体接入文档解析、文本切分、向量化与 RAG 检索等功能预留了接口基础。

Day02
DocFusion-AI 项目开发日志
今天主要完成了后端基础环境的搭建，并把文件上传和任务查询这一条最基本的功能链路跑通。
首先熟悉了项目的后端目录结构，确认了 backend/app 下各个模块的作用，比如 api 负责接口、db 负责数据库、services 负责具体逻辑处理等。同时确认了项目启动方式，需要在 backend 目录下运行 uvicorn app.main:app --reload 才能正常启动服务。
随后设计并完善了数据库中的 Task 模型，在 models.py 中定义了任务表，包含文件名、文件路径、文件类型、任务状态以及创建时间等字段。启动服务后，SQLite 数据库文件 app.db 已成功生成。
在接口部分，完成并调通了文件上传接口 /upload。上传文件后，后端会将文件保存到 uploads 目录，并在数据库中创建一条任务记录，同时返回对应的 task_id。
接着测试了任务查询接口 /tasks/{task_id}，可以根据返回的任务 ID 查询到对应任务的文件信息和当前状态。至此，已经实现了从文件上传、文件保存、写入数据库到根据任务 ID 查询任务信息的完整流程。
目前后端最基础的功能已经可以正常运行，后续计划根据项目需要继续完善任务状态管理，并为文档解析、文本切分等功能预留数据结构和接口。
